Title:

feat: 规划通用远程浏览器 Agent，支持客户现场 DOM 感知、大模型持续交互、受控自动化与双向文件传输

Description:

设计一套面向客户真实浏览器环境的通用 Chrome Extension 与远程 Agent 基础设施，用于解决客户系统依赖特定账号、权限、企业网络、登录状态、页面数据及动态 DOM，开发者无法在本地完整复现，因而难以远程分析问题、调试页面或开发定制自动化工具的问题。

客户可通过 Chrome 扩展程序开发者模式加载该通用插件，在明确授权后，使插件主动通过加密的跨网络连接接入开发者部署的 Agent Gateway。开发者电脑上的 Codex、大模型 Agent、IDE 工具或 MCP Client 可将客户当前浏览器作为远程工具使用，在不复制客户账号和登录凭据的情况下，直接观察、诊断和操作客户真实页面环境。

核心能力包括：

* 读取当前授权标签页的 URL、标题、视口、滚动位置、焦点、选区及页面加载状态。
* 采集语义 DOM、真实表单属性、可见性、坐标、关键样式、表格、虚拟列表、对话框、媒体、开放 Shadow DOM 和可访问 iframe 状态。
* 为页面元素分配稳定的会话级 nodeId，避免大模型直接依赖脆弱的 CSS Selector 或 XPath。
* 支持当前视口、选中元素、交互元素、任务相关节点及完整页面等不同上下文采集范围。
* 支持页面截图，并建立截图坐标和 DOM 节点之间的关联。
* 监听 DOM、表单、焦点、滚动、SPA 路由、对话框、网络错误及控制台错误等动态变化。
* 将高频底层变化防抖、合并并转换为可供大模型理解的语义事件。
* 支持全量页面快照、增量快照及快照差异查询。

将插件设计为可持续通信的远程浏览器 Agent，而不是一次性的“读取 DOM—请求模型—返回结果”流程。任务执行期间，大模型可随时通过 MCP 或工具协议调用客户浏览器，包括：

* 获取当前页面摘要和最新 DOM 快照。
* 查询指定元素、表单或表格状态。
* 获取快照差异。
* 获取截图、控制台错误和失败网络请求。
* 等待指定页面状态。
* 订阅或取消订阅页面事件。
* 执行受控浏览器操作。
* 请求客户人工介入。
* 向客户发送文件或请求客户选择文件。
* 暂停、恢复或终止远程任务。

通信架构由以下组件组成：

1. Remote Browser Chrome Extension
   安装在客户浏览器中，负责页面感知、动作执行、状态推送、客户确认、文件收发和本地日志。

2. Agent Gateway
   负责跨网络连接、身份认证、会话路由、消息转发、事件缓存、文件传输、断线恢复和审计日志。

3. Remote Browser MCP Server
   将客户浏览器暴露为 Codex 或其他大模型可调用的标准工具。

4. Developer Agent Client
   运行在开发者电脑上，连接 Gateway、暴露远程浏览器能力、读取开发者主动选择的本地文件，并接收客户返回的数据和文件。

5. Optional Local Agent Host
   通过 Chrome Native Messaging 提供普通扩展无法完成的本地目录、CLI、本地模型、数据库、企业内网接口及长时间后台任务能力。

客户插件主动建立 WSS 出站连接，客户电脑无需开放入站端口。通信层支持：

* 一次性配对码。
* 开发者、客户及浏览器实例身份认证。
* 会话级访问令牌。
* 心跳、超时和指数退避重连。
* 请求 ID、命令 ID、事件 ID 和订阅 ID。
* 消息确认、顺序号、幂等键及重复消息过滤。
* 防重放 nonce 和命令有效期。
* 断线期间事件缓存。
* 重连后的状态补偿和任务恢复。
* 多客户、多浏览器、多窗口、多标签页及多 iframe 隔离。

考虑 Manifest V3 Service Worker 可能被暂停或重新启动，任务状态、标签页绑定、待处理调用、事件订阅、页面版本及最后确认消息不得只保存在内存中，需要通过 chrome.storage.session、chrome.storage.local 或 IndexedDB 持久化并恢复。

为每个远程页面维护 browserInstanceId、windowId、tabId、frameId、documentId、snapshotId 和 pageRevision。所有远程动作必须声明 basedOnRevision；如果客户页面已发生变化，插件应拒绝基于旧状态的危险操作，并返回 STALE_PAGE_STATE，要求模型重新读取页面后再规划。

自动化采用“观察—规划—校验—执行—重新观察—验证”的多轮闭环，每轮优先执行少量动作，避免模型一次生成长链路后盲目执行。动作失败时记录具体原因，尝试重新定位元素、有限次数重试，并将最新状态交还模型重新分析。

远程执行不得支持任意 JavaScript、eval、Function、远程脚本地址或动态代码执行。大模型和服务器只能发送符合 JSON Schema 的结构化白名单动作，由扩展本地预置执行器完成，包括：

* query
* inspect
* click
* doubleClick
* focus
* blur
* setValue
* typeText
* clearValue
* selectOption
* check
* uncheck
* scroll
* scrollIntoView
* hover
* pressKey
* uploadFile
* wait
* waitForElement
* waitForText
* waitForNavigation
* readText
* readTable
* extractData
* takeScreenshot
* verify
* goBack
* goForward
* reload

支持四种远程工作模式：

1. 远程观察模式
   只允许读取页面状态、DOM、截图、控制台和网络错误，不修改页面。

2. 远程协助模式
   大模型提出操作，由客户逐次确认后执行。

3. 受控自动化模式
   客户预先授权指定网站、标签页和动作范围，模型可在权限范围内自动操作。

4. 客户专用工具模式
   加载经过审核的站点适配器和工作流，执行固定的客户业务功能。

任务状态包括 CREATED、PLANNING、OBSERVING、EXECUTING、VERIFYING、WAITING_PAGE、WAITING_USER、PAUSED、COMPLETED、FAILED 和 CANCELLED。

遇到登录、验证码、支付、电子签名、敏感信息、主观判断或不可逆操作时，任务进入 WAITING_USER，由客户接管。插件监听恢复条件，客户处理完成后自动把最新页面状态推送给模型继续执行。

远程动作按照风险分级：

* read：页面读取类操作，可自动执行。
* safe-write：填写普通草稿、滚动、展开等低风险操作。
* sensitive：上传文件、修改关键字段等，需要客户确认。
* irreversible：提交、支付、发送、发布、删除、修改账号或权限等，每次必须确认。

远程会话期间，客户浏览器持续显示：

* 当前连接的开发者。
* 当前被访问的网站和标签页。
* 最近一次页面读取或执行动作。
* 当前文件传输状态。
* 完整日志入口。
* 暂停和立即断开按钮。

默认禁止读取或传输：

* 密码字段真实值。
* Cookie。
* Authorization Header。
* Access Token。
* API Key。
* 浏览器保存的密码。
* 信用卡号和 CVV。
* 未经客户选择的本地文件。
* 未授权的其他标签页。
* 站点适配器标记的敏感区域。

密码字段只允许返回类型、是否已填写及长度，不返回真实内容。网页文本、DOM 属性和接口数据均被视为不可信输入，不得覆盖系统规则、用户要求、权限策略或动作白名单，以防止网页提示词注入。

文件传输不通过大模型上下文承载二进制内容。大模型只发出 push_file、request_remote_file 等工具调用，实际文件由 Developer Agent Client、Agent Gateway、临时对象存储或专用二进制通道传输。

向客户发送文件时支持：

* 文件名、MIME 类型和文件大小。
* SHA-256 完整性校验。
* 分块传输和断点续传。
* 临时下载地址和有效期。
* 客户确认。
* 保存到 Downloads。
* 暂存到插件 IndexedDB。
* 交由站点适配器尝试上传到当前页面。

客户向开发者发送文件时，必须由客户主动选择具体文件。默认不得浏览客户硬盘、搜索任意目录或静默读取下载目录。任意本地路径读写和系统级操作由可选 Local Agent Host 完成。

将该通用插件同时建设为客户自动化开发平台，而不是针对每个客户复制整套代码。系统拆分为：

* Browser Agent Core：页面感知、通信、任务、权限、执行、验证、安全、日志和恢复。
* Capability Registry：统一注册通用能力和业务能力，并声明输入输出 Schema、权限要求及风险等级。
* Site Adapter SDK：实现页面识别、字段语义、元素定位、特殊控件处理、页面稳定条件、业务动作和校验规则。
* Workflow SDK：通过受限 JSON、YAML 或 DSL 定义输入、步骤、条件、循环、等待、人工确认、错误分支和输出。
* Customer Package：包含客户品牌、站点适配器、工作流、专用能力、风险策略、提示词和测试用例。

允许动态下发的声明式能力包括：

* URL 和页面匹配规则。
* 页面识别规则。
* 字段映射。
* 元素定位规则。
* JSON Schema。
* 工作流步骤。
* 条件、循环和等待规则。
* 数据转换规则。
* UI 配置。
* 风险等级。
* 模型提示词和业务说明。

声明式配置只能由本地受限解释器执行，不得包含 eval、Function、远程 script、动态 WebAssembly 或其他任意代码执行入口。

涉及新算法、新浏览器权限、新页面主世界桥接、新复杂控件支持、新本地文件处理或其他可执行逻辑时，必须生成新版插件代码，经过静态检查、自动测试、人工审核和重新构建后，再由客户重新加载或升级。

形成完整的客户自动化开发闭环：

客户提出需求
→ 客户加载通用插件并开启远程协助
→ 大模型读取客户真实页面和 DOM 环境
→ 分析问题并复现业务流程
→ 生成站点适配器、字段映射和自动化工作流
→ 在客户真实环境中受控验证
→ 根据最新页面状态继续修正
→ 固化为自动化或半自动化客户工具
→ 必要时构建并向客户发送新版插件。

该方案最终将客户不可复制的真实浏览器环境转换为开发者和大模型能够受控访问、持续观察、远程调试和快速扩展的浏览器 Agent 运行环境。
