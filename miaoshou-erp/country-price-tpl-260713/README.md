# 申报价模板管理工具

一键将申报价模板批量套用到按国家区分的申报价表格中，避免运营人员每次手动逐个填写。

## 快速使用

1. 打开目标网页的申报价弹窗，**先手动滚动表格到底部再滚回顶部**（确保虚拟列表渲染过全部行）
2. `F12` 打开浏览器控制台（Console）
3. 粘贴 `country-price-template.js` 全部内容，回车
4. 脚本自动滚动收集全部国家，完成后弹出管理面板

## 功能

| 操作 | 说明 |
|------|------|
| **+ 新增模板** | 输入模板名后进入二维表格编辑，当前表格有哪些国家就列出哪些国家 |
| **📋 从当前表格创建** | 一键抓取当前页面上所有可见国家及其已填的申报价，保存为模板 |
| **编辑** | 修改已有模板的国家和价格，编辑时自动同步最新国家列表 |
| **套用** | 将模板中每个国家的申报价批量填入当前表格，**并启动持续监控** |
| **停止套用** | 点击面板顶部蓝色状态条中的"停止套用"按钮 |
| **删除** | 删除不需要的模板 |

### 套用模板后的持续监控

点击"套用"后，脚本会在后台持续工作：

- **自动填充新出现的行**：用户滚动原表格时，虚拟列表回收旧 DOM、创建新 DOM，脚本会立刻把新出现行的申报价填入
- **不覆盖用户手动编辑**：用户自己修改了某个国家的申报价后，该国家被记入黑名单，不会被模板再次覆盖
- **恢复同步**：如果用户把价格改回恰好等于模板值，该国家自动移出黑名单，重新恢复模板同步
- 面板顶部会显示蓝色状态条："🔄 正在套用: xxx模板（已禁止编辑 N 国）"

## 匹配逻辑

- 套用模板时按 `trim()` 后的国家名逐行遍历匹配，不依赖顺序
- 即使 UI 上国家名有空格、换行等脏字符，也能正确找到对应行
- 模板中存了但表格中没有的国家会被跳过，不会报错

## 存储

模板保存在浏览器 `localStorage` 中，关闭浏览器后不会丢失。不同域名之间隔离。

## 技术踩坑记录

### 坑 1：vue-recycle-scroller 虚拟滚动

**现象**：DOM 中始终只有 ~20 行 `<div>`，实际上表格有 27 个国家。用 `wrapper.children` 只能拿到 14-20 个，且每次滚动数值都在变。

**原因**：`vue-recycle-scroller` 是一个虚拟滚动组件。它只渲染当前视口附近的行（visible + buffer），其余行的 DOM 节点被回收复用。所以无法通过 DOM 子节点获取完整国家列表。

**尝试过的失败方案**：

| 方案 | 结果 | 原因 |
|------|------|------|
| `wrapper.__vue__.$data` | ✗ | Vue 3 生产模式剥离了所有内部属性，返回 null |
| `scrollerEl.__vnode.component` | ✗ | 同生产模式，不存在 |
| `window.__VUE_DEVTOOLS_GLOBAL_HOOK__` | ✗ | 该页面未安装 Vue DevTools，hook 不存在 |
| 同步 `scrollTop` 循环 + 死等渲染 | ✗ | 同步代码阻塞主线程，浏览器无法渲染新帧 |
| 异步 `setTimeout` 逐段滚动 | ✗ | scroller 的 scrollTop 设置后虚拟列表内部逻辑不响应 |

**最终方案**（由用户提出路线）：
- 使用用户提供的精确 DOM 选择器（`ROOT_SEL`），这个锚点始终正确
- **操作前手动滚动表格到底部**，让虚拟列表渲染过全部行，DOM 中会包含所有 27 行（含 translateY(-9999px) 缓冲区）
- 如果还不够，脚本内通过异步滚动自动补全

### 坑 2：DOM 节点回收导致套用失效

**现象**：点击"套用"把模板价格填入当前可见行后，用户一滚动表格，之前填入的价格全没了。

**原因**：虚拟滚动在滚出视口时回收 DOM 节点，滚回时用新节点重新渲染。原本填好价格的 `<input>` 被删除了。

**解决方案**：
- **MutationObserver** 监听 wrapper 的 `childList`、`characterData`、`subtree`
- 每次 DOM 发生变化，立刻扫描全部可见行，对符合模板的填入价格
- 用 `requestAnimationFrame` 层级确保不丢帧

### 坑 3：用户手动编辑被模板覆盖

**现象**：用户手动把某国价格从模板值改成自定义值后，滚动一下又变回模板值。

**解决方案**：
- 全局 `input` 事件委托（`addEventListener('input', ..., true)`）
- 用户手动编辑任何价格 input → 该国记入 `userEdited` 黑名单
- MutationObserver 填价时跳过黑名单中的国家
- 用户改回恰好等于模板值 → 自动移出黑名单

### 坑 4：Vue/Element UI 的 input 值设置

**现象**：直接 `input.value = 'xxx'` 后面保存时值消失。

**原因**：Vue 通过 getter/setter 代理了 `value` 属性，直接赋值不触发响应式更新。

**解决方案**：
```javascript
const nativeValueSetter = Object.getOwnPropertyDescriptor(
  window.HTMLInputElement.prototype, 'value'
).set;
nativeValueSetter.call(input, value);
input.dispatchEvent(new Event('input', { bubbles: true }));
input.dispatchEvent(new Event('change', { bubbles: true }));
```

### 坑 5：生产模式无法访问 Vue 数据

**现象**：所有 `__vue__`、`__vnode`、`$data`、`$props` 等内部属性全部为 null/undefined。

**原因**：该页面用 Vue 3 生产构建，所有调试接口被 tree-shaking 移除。这是安全措施，防止生产环境数据泄露。

**影响**：意味着这个工具完全无法从 Vue 数据模型读取完整国家列表，只能依赖 DOM。

## 注意事项

- 页面表格的 DOM 结构变化后需调整选择器（文件顶部 `ROOT_SEL`）
- 运行前务必手动滚一遍表格，确保虚拟列表渲染过所有行
- 持续套用模式会在后台运行，关闭弹窗或刷新页面后自动停止
- 套用大量国家时模板建议在新建弹窗时打开（此时所有行一次性渲染）
